# Анализ содержимого `FOSWLY/documentation/content`

## 1. Общая структура каталога `content`

Каталог `content/` — это исходники страниц VitePress (Markdown) для сайта документации VOT.
Основные разделы:
- `index.md` — главная страница (лендинг VOT);
- `guide/` — практическое руководство по использованию `vot.js` и связанных режимов;
- `ecosystem/` — обзор и описание связанных проектов (voice-over-translation, vot-cli, vot-backend и др.);
- `api-examples.md`, `markdown-examples.md` — примеры возможностей VitePress/Markdown (скорее служебные/демо);
- `public/` — статические ассеты (картинки/лого и т.п.).

---

## 2. Главная страница (`content/index.md`)

Ключевые моменты:
- Оформлена как `layout: home` с героем VOT, описанием и кнопками:
  - **Get Started** → `/guide` (основной вход в документацию);
  - **API References** → `https://foswly.github.io/vot.js/` (автогенерированный API-референс по исходникам `vot.js`).
- Подчёркиваются свойства библиотеки:
  - **Simple** — простой API, скрывающий сложную логику VOT;
  - **Lightweight** — минимум зависимостей, упор на производительность;
  - **ESM Compatible** — современные ES-модули;
  - **Type-safe** — весь код на TypeScript;
  - **Multi-runtime** — поддержка Bun, Node.js и современных браузеров;
  - **Powered by Community** — развитие за счёт комьюнити.
- Внизу — важный дисклеймер: библиотека не аффилирована с правообладателями (Яндекс и пр.), все права на оригинальный софт принадлежат им.

## 3. Раздел Guide (`content/guide`)

### 3.1. `guide/index.md` — Get Started

Содержит быстрый старт по трём направлениям: Node/Bun, Browser, Core.

#### 3.1.1. JS runtimes (`@vot.js/node`)

- Установка: `npm install @vot.js/node` (или `bun add`, `pnpm add`, `yarn add`).
- Основной пример:
  - импорт `VOTClient` и `getVideoData` из `@vot.js/node`;
  - вызов `getVideoData("<video_url>")` для получения `videoData` (детектирует сервис и ID);
  - вызов `client.translateVideo({ videoData })` и лог ответа.
- Подчёркивается, что `@vot.js/node` — основной вариант для серверных рантаймов (Node/Bun).

#### 3.1.2. Browser (`@vot.js/ext`)

- Установка: `@vot.js/ext` аналогично Node-варианту.
- Важные отличия:
  - `getVideoData` в браузерной версии **принимает сервис**, а не ссылку; сам URL берётся из `window.location.href`;
  - нужно заранее определить сервис (например, `getService()`), иначе бросается ошибка.
- Пример показывает diff: замена импортов на `@vot.js/ext`, добавление логики получения сервиса и вызов `getVideoData(service[0])`.

#### 3.1.3. Core (`@vot.js/core` + `@vot.js/shared`)

- Установка: `@vot.js/core @vot.js/shared`.
- Описание:
  - в отличие от `@vot.js/node`/`@vot.js/ext`, **нет хелперов** типа `getVideoData` и готовой логики детекта сервисов;
  - предназначено для написания своих обёрток/клиентов и глубоких кастомизаций.

### 3.2. `guide/using-vot-worker.md` — Использование с VOT Worker

- Решает проблемы блокировок Яндекса/CORS: вместо прямого обращения к Yandex VOT используется прокси-сервер [`vot-worker`](https://github.com/FOSWLY/vot-worker).
- Возможности:
  - использовать публичный `vot-worker.toil.cc` или разворачивать свой.
- Код-пример:
  - замена `VOTClient` на `VOTWorkerClient` из `@vot.js/node`;
  - указание `host: "vot-worker.toil.cc"` (по умолчанию);
  - дальше использование идентично обычному клиенту.

### 3.3. `guide/get-subtitles.md` — Получение субтитров

- Объясняет, что субтитры генерируются **вместе** с переводом:
  - если сразу при запросе их нет, скорее всего появятся после завершения перевода.
- Пример использования:

```ts
const subs = await client.getSubtitles({
  videoData,
  requestLang: "ru",
});
```

- Логика: `getSubtitles` возвращает список субтитров (оригинальные и переведённые), формат которых описан в типах `vot.js`.

### 3.4. `guide/translating-stream.md` — Перевод стримов

- Описывает работу со стримами YouTube/Twitch (по состоянию на апрель 2025):
  - сервер возвращает **M3U8 аудиопоток**, а не `.mp3`;
  - нужно периодически опрашивать `translateStream` и затем вызывать `pingStream`, чтобы стрим не остановился.
- Пример кода:
  - цикл ожидания статуса `TRANSLATING` (interval=10 секунд), с ретраями через `setTimeout`;
  - при `STREAMING` — получение `result.url` и `pingId`;
  - запуск `setInterval` с `client.pingStream({ pingId })` каждые `response.interval * 1000`.
- Есть защита от случая, когда ожидание прерывается (`isAbortedWaitingStream`).

### 3.5. `guide/advanced-translation.md` — Продвинутая настройка перевода

Разбирает дополнительные поля и сценарии:

1. **Поддерживаемые сайты**:
   - перечисление через enum `VideoService`; также поддерживаются прямые ссылки на `.webm`/`.mp4`;
   - если сайт не поддерживается напрямую, можно написать "Helper", который достанет прямую ссылку и отдаст её на перевод.

2. **translationHelp**:
   - поле, позволяющее указать ссылки на **файл видео** и **файл субтитров** (`video_file_url`, `subtitles_file_url`);
   - критично, если `videoData.url` — не поддерживаемый сайт и не прямая ссылка на mp4/webm;
   - даёт возможность перевести видео по временной ссылке и затем использовать "нестандартный" URL без повторной передачи help.

3. **Translation options**:
   - `shouldSendFailedAudio` — поведение при изменении логики Яндекса для YouTube (отправка аудио-файла в байтах); рекомендуется оставлять `true`, если нет кастомной реализации;
   - `extraOpts.forceSourceLang` — пользователем выбран исходный язык (vs автоопределение);
   - `extraOpts.wasStream` — применимо только к YouTube; указывает, что видео раньше было стримом;
   - `extraOpts.bypassCache` — обход кеша перевода (нестабильно);
   - `extraOpts.useLivelyVoice` — включение режима Lively Voice (голосовое клонирование, требует авторизации);
   - `extraOpts.videoTitle` — заголовок YouTube-видео (пока неясно, влияет ли).

4. **Custom VideoService**:
   - возможность расширить/переопределить список сервисов через свой enum/union и параметр типа у `VOTClient<V>()`;
   - замечание: `host` и `videoId` важны в первую очередь для VOT Backend.

5. **Custom VideoService Helper**:
   - рекомендуется создавать хелперы как подклассы `BaseHelper` с методами `getVideoId` (обязателен) и `getVideoData` (опционален, если `needExtraData` истинно);
   - подробно описана схема `data/sites.ts` с флагом `needExtraData` и типами `MinimalVideoData`/`VideoData`.

### 3.6. `guide/translation-with-lively-voice.md` — Режим Lively Voice

- Описывает режим "живых голосов" и его ограничения:
  - требуется **дополнительная авторизация** (Яндекс-аккаунт);
  - поддерживается только пара `en → ru`;
  - возможно наличие других ограничений.

- Два способа авторизации:
  1. **OAuth токен** (`apiToken` в опциях клиента или `Authorization: OAuth ...` в `headers`).
  2. **Cookie Session_id** — оригинальный метод Яндекс.Браузера, cookie вида `Session_id=3:xxxx...` (не `noauth:...`).

- Использование:

```ts
await client.translateVideo({
  // ...
  extraOpts: { useLivelyVoice: true },
  // либо передача Cookie в headers
});
```

### 3.7. `guide/custom-fetch.md` — Кастомный fetch и прокси

- Подчёркивается, что `vot.js` изначально проектировался с учётом GM_fetch/GM_xmlhttpRequest и совместим с fetch-совместимыми клиентами (ofetch, GM_fetch и т.п.).
- Два уровня кастомизации:

1. **Своя fetch-функция (`fetchFn`)**:
   - можно передать при создании `VOTClient` и/или при вызове `getVideoData`;
   - пример с `ofetch.native`.

2. **Дополнительные опции `fetchOpts`**:
   - передаются при создании клиента и мержатся с внутренними настройками;
   - пример добавления заголовка `X-Test-Header`.

- Дополнительно описан **VOT Agent (Proxy)** (для Node):
   - `VOTAgent` — дефолтный `dispatcher` на базе Undici, убирает нежелательные стандартные заголовки Node;
   - `VOTProxyAgent` — всё то же плюс поддержка HTTP(S) прокси (на базе Undici ProxyAgent);
   - в Bun это не требуется.

## 4. Раздел Ecosystem (`content/ecosystem`)

### 4.1. `ecosystem/index.md` — Обзор экосистемы

Ключевые положения:
- **Official tools**:
  - `voice-over-translation` — userscript для озвучки и субтитров на множестве сайтов;
  - `vot-cli` — кроссплатформенный CLI для перевода и получения субтитров;
  - `vot-backend` — сервер для расширения списка поддерживаемых сайтов (м3u8 и др.), использующий `vot.js`.
- **Related official tools** (не используют `vot.js` напрямую, но помогают):
  - `vot-worker` — proxy-сервер для обхода блокировок/ CORS к Yandex VOT API;
  - `media-proxy` — proxy-сервис, помогающий использовать mp4/webm с ограниченных сайтов.
- Community tools — открытое приглашение добавлять свои инструменты через "Edit this page".

### 4.2. `ecosystem/voice-over-translation.md`

- Описание основного userscript-а: добавляет озвучку, переведённые и AI-субтитры на множество сайтов (YouTube, курсы и т.п.).
- Функции (кратко):
  - перевод видео в русский/английский/казахский;
  - перевод стримов YouTube (с задержкой);
  - AI-субтитры, оригинальные субтитры сайтов;
  - сохранение субтитров (`.srt`, `.vtt`, `.json`) и аудио (`.mp3`);
  - авто-перевод при открытии видео;
  - раздельная громкость оригинала и перевода, автоподстройка громкости;
  - горячие клавиши, кастомизация вида субтитров через CSS-переменные.

### 4.3. `ecosystem/vot-cli.md`

- CLI-инструмент для перевода видео и получения сабов/аудио в терминале.
- Возможности:
  - перевод по ссылке, сохранение субтитров (`.srt`, `.vtt`, `.json`) и аудио `.mp3`;
  - получение ссылки на файлы вместо скачивания (если не используются кастомные форматы сабов);
  - поддержка Lively Voices (анонс в `2.0.0`);
  - поддержка HTTP(S) прокси.
- Пример использования: `vot-cli [options] <link> [link2] ...`.

### 4.4. `ecosystem/vot-backend.md`

- Сервер для расширения списка поддерживаемых `vot.js` сайтов, особенно с м3u8-потоками:
  - поддержка Mux (Patreon), Reddit, Kodik, Kick, Apple Developers, EpicGames, ArtStation и др.;
  - можно использовать как из `vot.js`, так и отдельно, делая HTTP JSON-запросы.
